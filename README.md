# Access Pass System Analysis Case

## Problem
Заявки на пропуск подаются через разные каналы (мессенджеры, почта), 
теряются, нет единого статуса и прозрачности согласования.

Это приводит к:
- задержкам в выдаче пропусков
- ошибкам в данных
- высокой нагрузке на администратора

## Goal
Создать централизованную систему управления заявками на пропуск, которая:

- сокращает время обработки заявки
- делает процесс прозрачным для всех участников
- снижает количество ошибок

## System Overview
Система включает:

- фронт (форма подачи заявки)
- backend (обработка и логика)
- модуль согласования
- уведомления

Основной поток:
Employee → Manager → Pass Admin → Issued


## Roles

- **Employee** — создаёт заявку
- **Manager** — согласует или отклоняет
- **Pass Administrator** — выпускает пропуск


## Scope
- создание заявки
- согласование
- изменение статуса
- уведомления
